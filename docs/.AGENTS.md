# docs/ Directory Index

How the codebase is built. Agent-facing files are concise and LLM-optimized.

## Agent-facing (flat in `docs/`)

| File              | Purpose                                          | When to update                                   |
| ----------------- | ------------------------------------------------ | ------------------------------------------------ |
| `architecture.md` | System layers, key patterns, critical file paths | When adding new layers or changing core patterns |
| `conventions.md`  | Code patterns (DAL, withAuth, SWR, hooks)        | When establishing new project-wide patterns      |
| `data-model.md`   | DB semantics, JSONB types, relationships, RPCs   | After migrations with semantic meaning           |
| `services.md`     | External service reference table                 | When adding/changing third-party services        |

## Human-only (`runbooks/`)

Setup guides and operational docs. Never loaded into agent context.

| File                    | Purpose                          |
| ----------------------- | -------------------------------- |
| `capacitor-android.md`  | Android native shell setup       |
| `mcp-setup.md`          | MCP server configuration for IDE |
| `setup-google-cloud.md` | Google Cloud console setup       |
| `setup-sentry.md`       | Sentry dashboard setup           |
| `setup-supabase.md`     | Supabase dashboard setup         |
| `vercel-deployment.md`  | Deployment guide and env var ref |

## Conventions

- Target audience for agent-facing files: LLMs and developers. Favor tables over prose.
- Keep files under 250 lines. If growing beyond that, split or trim.
- Don't duplicate what's recoverable from source (DB schema, code itself).
